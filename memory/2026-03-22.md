## Dashboard Date Format Error - RESOLVED

**Issue:** HTML5 date input showing "The string did not match the expected pattern" error, specifically affecting the 每日热冷股筛选器 (Daily Hot/Cold Screener).

**Root Cause:** React's initial render passes empty string `''` to `type="date"` input before `useEffect` sets the actual date value. HTML5 date inputs validate the value immediately and reject empty strings.

**Final Solution:** Conditional rendering approach:
- If `date` is empty → render `type="text"` placeholder ("加载中..." / "Loading...")
- If `date` has value → render `type="date"` with actual value

**Applied to:**
- CheckStockModal
- RunScreenerModal
- ResultsView

**Files Modified:**
- `dashboard2/frontend/src/App.tsx`
